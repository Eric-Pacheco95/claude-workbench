# Lessons Learned

> Lightweight retrospective log. Add an entry after each sprint, project milestone, or significant incident.
> Claude reads this directory when starting work on similar problems and surfaces relevant lessons.

## Format

Create one file per sprint/milestone: `YYYY-MM-DD_[project]-sprint-N.md`

```markdown
# Lessons Learned — [Project] Sprint N / [Milestone]

**Date:** YYYY-MM-DD
**Project:** [Project name]
**Type:** Sprint | Milestone | Incident | Discovery

## What Went Well
- [Specific practice or decision that worked — be concrete enough to repeat]

## What Didn't Work
- [Specific thing that caused friction, delay, or rework — be specific enough to avoid]

## What To Do Differently
- [Actionable change for next time]

## Surprising Findings
- [Things that were unexpected — positive or negative]

## Process Notes
- [Anything about the workflow, tooling, or collaboration that's worth noting]
```

## Active Lessons

> Claude: when starting a new project, scan this directory for lessons from similar projects and surface relevant ones to the user before PLAN phase begins.
